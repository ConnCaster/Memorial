# Мемориал
Инструмент для сопоставления персональных данных военнослужащих с оцифрованными архивными ведомостями 
о безвозвратных потерях периода ВОВ.
Работает с данными формата ОБД «Мемориал».

### Проблема
Исследователь имеет список интересующих его военнослужащих, представляющий собой паспорт какого-либо конкретного захоронения (в Excel)
и хочет найти, в каких архивных документах-ведомостях они упоминаются.
Ведомость - документ, который отражает отчет конкретного воинского подразделения о потерях в определенный период времен.

### Решение
Memorial загружает ведомости в MongoDB, затем для каждого солдата из «паспорта» выполняет нечёткий поиск по всем документам, учитывая:
- вариативность написания ФИО и географических названий
- сокращения воинских званий
- неполные или разнородные форматы дат


## Загрузка ведомостей в MongoDB
Правильные ведомости, которые можно использовать для загрузки в БД, лежат в директории  `input/ved_correct`.
Для загрузки их в БД нужно запустить программу с флагом -u:
```bash
./Memorial -u "/home/user/dir/programming/C++/Yaroslava/Memorial/input/ved_correct"
```
При этом будут обработаны все файлы в директории `input/ved_correct`(xlsx-формат)

## Поиск по паспорту
Для поиска используется файл паспорта захоронения. 
Задача - найти ведомость среди ведомостей, предварительно загруженных в БД, в которой больше всего совпадений людей с паспортом. 
Кроме того, вывести по каждому солдату из паспорта захоронения информацию о том, 
в какой ведомости (или в каких ведомостях, если их много), он встречался.

Для поиска запустим программу с флагом -p:
```bash
./Memorial -p "./input/pass_89503005 (1).xlsx"
```

### Критерии совпадения (SoldierEntry::operator==)
Солдаты считаются одним лицом, если выполняется одно из условий:
- Фамилия + Имя + Отчество + Дата рождения
- Фамилия + Отчество + Дата рождения + Дата выбытия  
- Фамилия + Дата рождения + Дата выбытия (минимальный набор)
